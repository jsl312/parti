# Parti — 건축 사전기획 리서치 스튜디오

사이트와 공간 유형만 입력하면 **리서치 → 문제정의 → 컨셉 구조화 → 컨셉 이미지**까지
이어서 진행하도록 도와주는 웹앱입니다. 로컬 AI(Ollama)와 로컬 이미지 생성(ComfyUI)을
사용해 비용 없이 돌릴 수 있습니다.

> 이 문서는 **비개발자**도 바로 실행하고 사용할 수 있도록 쓴 안내서입니다.
> 코드 폴더(`app`, `lib` 등)는 건드릴 필요가 없습니다.

---

## 1. 서버 실행하기 (가장 중요)

폴더 안의 실행 파일을 **더블클릭**하면 됩니다.

| 파일 | 언제 쓰나요 |
|---|---|
| **`서버 실행.bat`** | 평소에 서버를 켤 때 (대부분 이걸 씁니다) |
| **`업데이트 후 실행.bat`** | 개발자가 코드를 수정해 준 뒤, 변경사항을 반영해서 켤 때 |

실행하면 창 2개가 새로 열립니다:
- **Parti Web Server** — 웹서버 (사용 중에는 닫지 마세요)
- **ngrok tunnel** — 외부 접속용 터널 (사용 중에는 닫지 마세요)

접속 주소:
- **내 PC에서:** http://localhost:3000
- **외부(휴대폰·다른 사람)에서:** ngrok 창에 표시되는 `Forwarding https://...` 주소

종료하려면 열린 창 2개를 모두 닫으면 됩니다.

### 처음 한 번만 준비할 것
1. **Node.js** 설치 (https://nodejs.org — LTS 버전)
2. **ngrok** 설치 + 로그인 (`ngrok config add-authtoken <내토큰>`) — 외부 공개가 필요할 때만
3. **Ollama** 실행 + 모델 받기: `ollama pull qwen2.5:14b` (텍스트 생성용)
4. **ComfyUI** 실행 (이미지 생성용, 기본 주소 `http://localhost:8188`)

---

## 2. 외부 접속 주소 고정하기 (ngrok)

기본은 켤 때마다 ngrok 주소가 바뀝니다. **고정 주소**를 쓰려면:

1. ngrok 대시보드에서 발급받은 고정 도메인(예: `my-parti.ngrok-free.app`)을 복사
2. 이 폴더의 **`ngrok-domain.txt`** 를 열어 그 도메인 **한 줄만** 남기고 저장
3. 다음부터 `서버 실행.bat` 을 켜면 항상 같은 주소로 열립니다

> `ngrok-domain.txt` 가 비어 있거나 기본 안내문구 그대로면, 임시(랜덤) 주소로 열립니다.

---

## 3. 내가 만든 작업물은 어디에?

- **`parti-output/`** — "일괄 생성"으로 만든 컨셉·이미지가 저장되는 폴더입니다.
  `parti-output/<프로젝트>/<실행시각>/001, 002 …` 형태로 쌓이고, 각 폴더 안에
  `concept.md`(컨셉 정리), `prompts.md`(이미지 프롬프트), `images/`(생성된 이미지 8장)가 들어갑니다.
- **진행 중인 프로젝트 데이터**는 브라우저 안(localStorage·IndexedDB)에 저장됩니다.
  - 같은 브라우저로 다시 들어오면 그대로 이어집니다.
  - 백업/이전은 앱 안의 **내보내기/가져오기(.json)** 기능을 사용하세요.

---

## 4. 설정 바꾸기

웹앱 우상단 **설정** 페이지에서:
- **LLM(텍스트)**: Ollama / Anthropic / Gemini 선택. 기본 `Ollama + qwen2.5:14b`.
- **이미지**: ComfyUI / OpenAI / Gemini. 로컬은 **ComfyUI**.
  - 모델 종류에서 **Z-Image Turbo** 선택 시 전용 파일 3개(diffusion·텍스트 인코더·VAE) 지정.
- **일괄 생성**: 결과를 저장할 출력 폴더 경로(기본 `./parti-output`).
- **V-World**: "주변 대지 분석"에 쓰는 인증키.

AI 작업 화면마다 모델 드롭다운이 있어, 설정에 등록해 둔 모델 중 그 자리에서 골라 쓸 수 있습니다.
모든 키는 **브라우저에만** 저장됩니다.

## 5. 폴더 안내 (참고용 — 건드리지 않아도 됩니다)

| 폴더/파일 | 내용 |
|---|---|
| `서버 실행.bat` / `업데이트 후 실행.bat` | 서버+ngrok 실행 파일 |
| `ngrok-domain.txt` | ngrok 고정 도메인 설정 |
| `parti-output/` | **내가 생성한 컨셉·이미지 결과물** |
| `app/` | 화면(페이지)과 서버 기능 코드 — 프로그램 본체 |
| `lib/` | 내부 로직(AI 연동, 이미지 생성, 저장 등) |
| `components/` | 공용 UI 컴포넌트 |
| `public/` | 아이콘 등 정적 파일 |
| `package.json`, `next.config.ts`, `tsconfig.json` | 프로그램 설정 파일 |
| `AGENTS.md`, `CLAUDE.md` | 개발/AI 보조용 메모 (무시해도 됩니다) |
| `node_modules/`, `.next/` | 자동 생성 폴더 (직접 만지지 마세요) |

## 6. 자주 묻는 문제

- **이미지가 안 만들어져요** → ComfyUI 창이 켜져 있는지, 설정의 ComfyUI 주소/모델이 맞는지 확인.
- **텍스트(컨셉)가 이상하게 반복돼요** → Ollama 모델을 `qwen2.5:14b` 로 사용하세요(가장 안정적).
- **외부 주소로 안 열려요** → ngrok 창이 켜져 있고 로그인(authtoken)이 되어 있는지 확인.
- **VRAM 부족(OOM)** → 16GB에서 텍스트(qwen)와 이미지(ComfyUI)가 동시에 메모리를 씁니다.
  Z-Image는 **fp8** 파일을 쓰고, 필요하면 `OLLAMA_KEEP_ALIVE=0` 또는 더 작은 텍스트 모델(`qwen2.5:7b`)을 쓰세요.
- **첫 실행이 느려요** → 처음엔 빌드가 한 번 돌아갑니다(수십 초). 이후엔 빨라집니다.
